# TODO Web Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a portfolio-quality full-stack TODO web app with a marketing landing page, credential auth, PostgreSQL persistence, and a polished card-based dashboard.

**Architecture:** Use `Next.js App Router` as a single full-stack application with route groups for marketing, auth, and the protected app area. Keep authentication, database access, and TODO business logic in focused modules so UI components stay presentation-oriented and every server-side read/write is scoped to the current user.

**Tech Stack:** `Next.js`, `React`, `TypeScript`, `Tailwind CSS`, `shadcn/ui`, `PostgreSQL`, `Prisma`, `Auth.js`, `Zod`, `Vitest`, `React Testing Library`, `Playwright`

```ts
Credentials({
  credentials: { email: {}, password: {} },
  async authorize(credentials) {
    const user = await prisma.user.findUnique({ where: { email: credentials.email } });
    if (!user) return null;
    const valid = await bcrypt.compare(credentials.password, user.passwordHash);
    return valid ? { id: user.id, email: user.email, name: user.name } : null;
  }
})
```

```ts
export async function listTodosForUser(userId: string, filters?: TodoFilters) {
  return prisma.todo.findMany({
    where: {
      userId,
      ...(filters?.status && filters.status !== 'all' ? { status: filters.status } : {}),
      ...(filters?.priority && filters.priority !== 'all' ? { priority: filters.priority } : {})
    },
    orderBy: [{ status: 'asc' }, { dueDate: 'asc' }, { createdAt: 'desc' }]
  });
}
```

## Notes for the Implementer

- Use `pnpm` consistently for dependency management.
- Generate only the `shadcn/ui` primitives actually used in the interface.
- Prefer server components for dashboard reads and server actions for mutations.
- Keep all Prisma access in `features/todos/repository.ts` or `lib/db.ts` helpers; do not scatter raw queries across page files.
- Any server action that mutates TODOs must read the authenticated session and scope writes by `session.user.id`.
- If the project is initialized as a git repository before implementation starts, preserve the commit cadence suggested above. If git is still unavailable, treat each task boundary as a review checkpoint.
